## ⛳ 황금 규칙
> **`audit` 점검을 통과해야만 master에 머지된다.** master 직접 push는 막혀 있다 — 무조건 PR.

---

## 🔁 매번 쓰는 명령어

```bash
# 1) 브랜치 생성
git checkout master && git pull
git checkout -b feature/<단계>-<내용>

# 2) deliverables/ 안에서 산출물 작성

# 3) ★ 커밋 전 로컬 점검 ★
bash scripts/pre-submit.sh            # 전체
bash scripts/pre-submit.sh SD22-2     # 특정 단계만 (빠름)

# 4) 통과(✅) 후 커밋·푸시
git add . && git commit -m "docs(...): ..." && git push -u origin <브랜치>

# 5) PR → audit ✅ → 머지
gh pr create --base master --fill
```

엔진만 단독 실행:
```bash
python audit_engine.py --rules rules/audit_rules.yaml [--phase SD21-2]
```

## 🔎 점검이 보는 4가지

| 유형 | 통과 조건 | 실패 예 |
|------|-----------|---------|
| 존재 | 단계별 필수 산출물 파일이 있음 | 비전기술서 누락 |
| 내용 | 검토항목별 핵심 키워드 포함 | 연계설계서에 `연계 주기` 없음 |
| 형식 | 금지 토큰 없음 + 필수 섹션 있음 | `TBD` 잔존, `목차` 삭제 |
| 추적성 | RTM 행이 끊기지 않음 | REQ→UC→CLS→코드→시험 단절 |

**금지 토큰:** `TBD` `T.B.D` `미정` `추후작성` `작성예정` `XXX` `???`
**필수 섹션(모든 문서):** `개정이력` `목차`

## ⚙️ 룰 추가/수정
`rules/audit_rules.yaml` **한 파일만** 수정 → 로컬 점검으로 확인 → PR.
엔진(`audit_engine.py`)은 건드리지 않는다.

`check.type`: `exists` · `contains` · `not_contains` · `traceability` · `min_count`

## 🆘 자주 막히는 것
- **master push 거부** → 정상. PR로.
- **"산출물 없음"** → 파일명이 룰셋 `pattern`과 맞는지 확인.
- **docx/pdf 에러** → `pip install -r requirements.txt`.
- **RTM 통과 안 됨** → 새 REQ/UC/CLS/CMP/UT/UAT 생길 때마다 `deliverables/RTM.csv` 행 채우기.
- **실패 원인 보기** → 점검 후 생성되는 `audit_report.md`.
