# 신입 첫 과제 — 비전기술서 보완 PR 머지하기

> [ONBOARDING.md](../ONBOARDING.md)의 실습(STEP 1~6)을 마친 분을 위한 **실전 첫 과제**입니다.
> 목표: 실제 산출물을 보완해 **PR 하나를 직접 master까지 머지**해 봅니다. (예상 30분)
>
> 멘토에게 이 문서 링크와 함께 "첫 과제 시작합니다" 라고 알려주세요. 멘토는 아래 체크박스로 진행을 확인합니다.

---

## 🎯 과제 개요

| 항목 | 내용 |
|------|------|
| **대상 산출물** | `deliverables/01_요구분석/비전기술서.md` |
| **할 일** | 비어 있는 섹션(`(...)` 플레이스홀더)을 실제 내용으로 채우고, 금지 토큰 없이 점검 통과시켜 PR 머지 |
| **관련 감리코드** | SD21-2 (요구분석 단계) |
| **완료 기준** | `audit` CI 체크 ✅ + 멘토 리뷰 승인 + master 머지 완료 |

## ✅ 진행 체크리스트

### 1단계 — 작업 브랜치 준비
- [ ] `git checkout master && git pull`
- [ ] `git checkout -b feature/first-task-비전기술서-<나의이름>`

### 2단계 — 비전기술서 채우기
`deliverables/01_요구분석/비전기술서.md`를 열어 아래를 **실제 내용으로** 작성합니다.
샘플 프로젝트(예: "지역 기후적응 정보 플랫폼")를 가정하고 자유롭게 채워도 됩니다.

- [ ] **개정이력** — 본인 이름과 오늘 날짜로 한 줄 추가 (`YYYY-MM-DD` 플레이스홀더 교체)
- [ ] **1. 사업 개요 및 목표** — 사업 배경·목표 2~3문장
- [ ] **3. 상위 수준 요구사항** — 핵심 요구사항 3건 이상
- [ ] **4. 제약사항** — 기술/일정/예산/법제도 중 2가지 이상
- [ ] 문서 전체에서 `TBD`, `미정`, `추후작성` 등 **금지 토큰이 없는지** 확인
- [ ] `개정이력`, `목차` 섹션이 그대로 남아 있는지 확인 (삭제 금지)

> 💡 SD21-2 단계 전체가 PASS하려면 다른 요구분석 산출물도 함께 충족돼야 하지만, 첫 과제에서는 **비전기술서만 손대고 단계 전체 점검이 깨지지 않는지** 확인하면 됩니다.

### 3단계 — 로컬 점검 (커밋 전 필수)
- [ ] `python audit_engine.py --rules rules/audit_rules.yaml --phase SD21-2`
- [ ] 맨 아래 `✅ 감리 제출 기준 충족` 확인 (FAIL이 있으면 `audit_report.md` 보고 보완)

### 4단계 — 커밋 & 푸시
- [ ] `git add deliverables/01_요구분석/비전기술서.md`
- [ ] `git commit -m "docs(sd21): 비전기술서 개요·요구사항·제약사항 작성"`
- [ ] `git push -u origin feature/first-task-비전기술서-<나의이름>`

### 5단계 — PR 생성 & CI 통과
- [ ] `gh pr create --base master --fill` (또는 GitHub 웹에서 PR 생성)
- [ ] PR 화면에서 **`audit` 체크가 ✅(초록불)** 인지 확인
- [ ] 빨간불(❌)이면 PR 코멘트의 실패 항목 확인 → 수정 → 다시 푸시

### 6단계 — 리뷰 & 머지
- [ ] 멘토에게 리뷰 요청
- [ ] 리뷰 승인 후 **Squash and merge**로 master 머지
- [ ] 머지된 PR 링크를 멘토/팀 채널에 공유 🎉
